### 8.1 프락시 [클라이언트와 서버 사이에 위치한 HTTP 중개자]

- 주로 보안을 위하여 요청과 응답을 필터링한다.

### 8.2 캐시 [자주 찾는 웹페이지를 클라이언트 가까이에 보관하는 HTTP 창고]

- HTTP는 캐시를 효율적으로 동작하게 하고 캐시된 콘텐츠를 최신 버전으로 유지하면서 동시에 프라이버시도 보호하는 기능을 한다.

### 8.3 게이트웨이 [다른 애플리케이션과 연결된 특별한 웹 서버]

- HTTP 트래픽을 다른 프로토콜로 변환하기 위해 사용
- 클라이언트는 게이트웨이와 통신하고 있음을 알아채지 못하는데, 게이트웨이가 진짜 서버인 것처럼 요청을 다루기 때문이다.

### 8.4 터널 [단순히 HTTP 통신을 전달하기만 하는 특별한 프락시]

- 비 HTTP 데이터를 하나 이상의 HTTP 연결을 통해 전송해주기 위해 사용된다.

### 8.5 에이전트 [자동화된 HTTP 요청을 만드는 준지능적 웹클라이언트]

- 스스로 웹을 돌아다니며 HTTP 트랜잭션을 일으키고 콘텐츠를 받아오며, 콘텐츠 보관소 혹은 데이터베이스를 만드는 클라이언트 프로그램
